# Система учёта оборудования - Asset Tracker

Полноценная веб-система для учёта компьютеров, ноутбуков, мониторов и другой техники.
Asset Tracker — это веб-приложение, предназначенное для удобного ведения учёта IT-оборудования. Оно позволяет не только хранить информацию об активах, но и отслеживать их перемещения и изменения характеристик со временем. Все изменения активов заносятся пользователем самостоятельно, для обеспечения простого взаимодействия с системой. Приложение предоставляет интуитивно понятный интерфейс, а также мощные инструменты импорта и экспорта данных через Excel.

## 🛠 Технологии
- Бэкенд: FastAPI (Python)
- Фронтенд: React
- БД: SQLite 
- ORM: SQLAlchemy
- Хранение паролей: Passlib[bcrypt]

## Основные функции
## Учёт активов
- Создание записей: Добавление новых IT-активов с подробной информацией (инвентарный/серийный номер, модель, тип, статус, местоположение, пользователь, даты, характеристики, комментарии и др.).
- Просмотр списка: Таблица со всеми активами, поддержка постраничного вывода.
- Редактирование: Изменение информации об уже существующих активах.
- Удаление: Возможность удалить запись об активе.
- Фильтрация: Отображение активов по типу оборудования (Мониторы, Компьютеры, Ноутбуки, Прочее).
- Поиск: Быстрый поиск по всем полям активов (инвентарный номер, серийный номер, модель, расположение и т.д.).
- История ремонтов: Просмотр историй ремонта, где, когда, у кого и что менялось, даты, имена сервисных центров
- Отслеживание возраста устройств: Возраст автоматически высчитывается, если указана дата покупки устройства, если нет, то можно вручную указать приблизительное время существования актива
- Скан QR-кодов: Если на складе много ПК и ноутбуков, под разные задачи, можно отсканировать QR код, который выведет модальное окно из приложения с информацией о внутренних комплектуюших



## Отслеживание истории изменений
- Автоматический аудит: Приложение автоматически фиксирует любые изменения ключевых полей актива (инвентарный номер, серийный номер, местоположение, пользователь, статус, модель, тип, характеристики и др.).
- Подробная информация: Для каждой записи истории отображается:
    - Пользователь: Кто внес изменение.
    - Поле: Какое именно поле было изменено.
    - Старое и новое значение: Что было до изменения и что установлено теперь.
    - Дата: Когда произошло изменение.
- История в Excel: При экспорте данных история изменений также включается в отдельный лист Excel-файла.
- Импорт истории: Возможность импортировать историю изменений из Excel-файла.

## Импорт и экспорт данных
- Экспорт в Excel: Экспорт всех отфильтрованных/найденных активов и их истории изменений в формат .xlsx. Позволяет легко создавать резервные копии или отчёты.
- Импорт из Excel: Импорт данных об активах и истории изменений из заранее подготовленного Excel-файла (формат файла должен соответствовать шаблону экспорта). Позволяет быстро заполнить базу или перенести данные.

## Управление пользователями и безопасность
- Аутентификация: Вход в систему по логину и паролю.
- Роли пользователей: Простая система ролей "Администратор" / "Обычный пользователь".
    - Администратор: Может создавать, редактировать, удалять активы, импортировать данные и управлять пользователями.
    - Обычный пользователь: Может просматривать список активов, фильтровать, искать, экспортировать данные.
- Хэширование паролей: Пароли пользователей надёжно хэшируются перед сохранением в базе данных.

## Статистика и отчёты
- Панель статистики: Отображение ключевых метрик на главной странице:
    - Общее количество активов.
    - Количество ноутбуков/компьютеров/мониторов и прочих активов.
    - Количество активов с истекающей гарантией (ближайшие 30 дней).
    - Количество активов, находящихся на ремонте.
    - Просмотр истории ремонтов активов.
- Отчёт о гарантиях: Отдельная вкладка с перечнем активов, у которых гарантия истекает в ближайшие 30 дней.

## Дополнительные возможности
- Очистка базы данных: Администратор может полностью очистить базу данных (с предупреждением и возможностью сделать резервную копию перед этим).
- Адаптивный дизайн: Интерфейс адаптирован для использования на различных устройствах.
- Генерация QR-кода: Активы с комплектующими (ПК и ноутбуки) имеют возможность сгенерировать QR-код, для быстрого просмотра характеристик, без необходимости ручного поиска актива в приложении.

## Настрой конфигурацию (.env файлы)

Перед запуском нужно создать два файла с настройками — по одному для фронтенда и бэкенда. Все настройки, включая IP-адрес сервера, указываются только здесь. В код лезть не нужно.

**Почему это важно:** фронтенд обращается к бэкенду по сети, и ему нужно знать адрес сервера. При сборке Vite автоматически подставляет IP из `.env` в код — менять `App.jsx` вручную не требуется. Если сменится IP сервера — меняешь только `.env` файлы и пересобираешь.

### frontend/.env

Создай файл `frontend/.env` со следующим содержимым:

```
VITE_API_URL=http://IP_ТВОЕГО_СЕРВЕРА:8000
```

Замени `IP_ТВОЕГО_СЕРВЕРА` на реальный IP. Например: `http://192.168.1.100:8000`

### backend/.env

Создай файл `backend/.env` со следующим содержимым:

```
SECRET_KEY=сгенерируй_случайную_строку_минимум_32_символа
TOKEN_EXPIRE_MINUTES=480
ALLOWED_ORIGINS=http://IP_ТВОЕГО_СЕРВЕРА:5173,http://IP_ТВОЕГО_СЕРВЕРА:8000
```

`SECRET_KEY` — секретный ключ для подписи JWT-токенов. Сгенерировать можно командой:

```
openssl rand -hex 32
```

`ALLOWED_ORIGINS` — адреса, с которых браузер может обращаться к API. Укажи тот же IP, что и в `frontend/.env`.

> Если сменился IP сервера — обнови оба `.env` файла и пересобери контейнеры командой `podman-compose up --build`. Код трогать не нужно.

## Как легко настроить backup'ы 

Бэкапы можно делать вручную из системы через кнопку "Экспорт", которая скачивает файл формата "Excel" со всей информацией об активах из БД, однако, такой способ не делает бэкап пользователей, поэтому если нужно более глубокое резервное копирование, можно настроить бэкапы на удаленный диск с помощью программы командной строки "Rclone". Она позволяет делать синхронизацию и копирование между локальным компьютеров (в нашем случае сервером) и более чем 40 облачными сервисами (Google Drive, Dropbox, Amazon S3, Yandex Disk и т.д.).


## Установка rclone, более подробно можно прочитать на официальном сайте https://rclone.org/install/
```
# Для начала убедитесь, что есть программа для распаковки zip архивов на сервере, если нет:
sudo dnf install unzip  #RHEL
sudo apt install unzip  #Debian

# Команда установки rclone:
curl https://rclone.org/install.sh | sudo bash # Далее идет установка по документации на сайте rclone 

```

## Настройка резервного копирования в Google Drive после установки rclone

```
# 1. Создайте папку для локальных резервных копий:
mkdir -p /home/user/asset-tracker/backups

# 2. Убедитесь, что rclone настроен и может получить доступ к вашему диску:
rclone lsd название_вашего_хранилища_rclone: # Эта команда должна показать содержимое корня вашего удаленного хранилища без ошибок.

# 3.1. В папке scripts есть готовый скрипт с именем "backup_db.sh", необходимо отредактировать его, перейдите в него и измените переменные в блоке "Настройки" на свои.

# 3.2. Сделайте скрипт исполняемым:
chmod +x /home/user/asset-tracker/scripts/backup_db.sh

# 3.3. Протестируйте скрипт вручную:
/home/user/asset-tracker/scripts/backup_db.sh # Проверьте файл лога /home/server/asset-tracker/backups/backup.log и ваше облачное хранилище, чтобы убедиться, что всё работает.

# 4.1. Настройка автоматического запуска (cron):
crontab -e

# 4.2. Добавьте строку для планирования. Например, для ежедневного выполнения в 02:05 ночи:
5 2 * * * /home/user/asset-tracker/scripts/backup_db.sh

# 4.3. Проверьте, что задание добавлено:
crontab -l

# 5. Готово, резеврное копирование БД настроено на ваш Google Drive.

```
